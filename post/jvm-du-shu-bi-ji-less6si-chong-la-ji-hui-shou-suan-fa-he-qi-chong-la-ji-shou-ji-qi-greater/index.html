JVM读书笔记<6、四种垃圾回收算法和七种垃圾收集器>

Author: biu~ Date: 2019-10-07 Reading Time:3.4 mins words:956 Category: JVM Views: -

垃圾收集算法：

1、复制。2、标记清除。3、标记整理。4、分代收集

七种垃圾收集器

  1. Serial(复制)：串行收集器

    • 优点：单线程情况下，简单高效。
    • 缺点：
      • ①单线程进行垃圾收集
      • ②垃圾收集时需要暂停所有工作线程。
    • 适用场景：运行在Client模式下的虚拟机

  2. ParNew(复制)：Serial的多线程版本，可搭配CMS适用

    • 优点：GC时对CPU资源利用率高
    • 缺点：多个GC线程存在额外开销
    • 适用场景：运行在Server模式下的虚拟机

  3. Parallel Scavenge (复制)：关注吞吐量

    • 优点：
      • ①能够精确控制吞吐量(-XX：GCTimeRatio和-XX:MaxGCPauseMillis)
      • ②自适应调节优化策略。
    • 适用场景：没有太多交互的后台运算任务。

  4. Serial Old(标记整理)：Serial的老年代版本

    • 优缺点：同Serial
    • 适用场景：
      • ①运行在Client模式下的虚拟机。
      • ②CMS收集器出现Concurrent Mode Failure的后备解决方式

  5. Parallel Old(标记整理):Parallel Scavenge的老年代版本

    • 优缺点以及适用场景:同Parallel Scavenge

  6. CMS(标记清除)：Concurrent Mark Sweep。

    • 优点：并发收集，低停顿
    • 缺点：
      • ①并发标记和并发清除阶段占用CPU资源，会导致应用程序变慢。
      • ②无法处理浮动垃圾：即在并发清理阶段产生的新的垃圾无法在当次进行回收，只能下一次GC时 进行处理。
      • ③并发清除阶段需要预留内存给用户进程运行。若空间不足，则会出现Concurrent Mode
        Failure，虚拟机会采用后备的Serial Old进行垃圾收集，导致GC停顿时间变长。
      • ④标记清除算法会导致内存碎片，需要额外的full GC进行合并整理。
    • 过程：
      • ①初始标记：标记GC Roots能直接关联到的对象
      • ②并发标记:GC Roots tracing
      • ③重新标记：修正并发标记期间变动的标记
      • ④并发清除。
        【注】时长大小：并发标记/并发清除>重新标记>初始标记

7、G1(整体：标记整理，局部：复制)：
* 堆空间划分方式：将Java堆划分为多个大小相等的独立区域，仍然存在新生代和老年代的概念，但
是物理上不再连续。
* 优先级的区域回收方式：后台维护一个按垃圾堆积的价值大小排列的Region优先列表，每次都优先 回收价值最大的Region
* 避免全堆扫描的方式：一个Region对应一个Remembered Set，其中记录了被其他Region所引用的信
息，在GC根节点的枚举范围内加入Remembered Set即可避免全堆扫描
【注】能后建立可预测的停顿时间模型的原因：G1收集器对空间划分方式，G1收集器每次不是对全堆进行收集，而是使用优先级的区域回收方式。
* 优点：
* ①G1整体上基于标记整理，局部上基于复制算法，所以G1不会产生内存空间碎片。
* ②可预测的停顿时间模型：用户可以指定在某个时间片段内，消耗在GC上的时间不能超过设定的 上限。
* 过程：
* 1、初始标记：标记GC Roots能直接关联到的对象。
* 2、并发标记：GC Root Tracing，即对堆中对象做可达性分析。
* 3、最终标记：修正并发标记期间变动的标记，变动记录在Remembered Set Logs，并合并到 Remembered Set中。
* 4、筛选回收：对各个Region的回收价值和成本进行排序，根据用户所期望的GC停顿时间制定 回收计划。
【注】需要停顿工作线程:1.初始标记。2.最终标记(多个GC线程并行)。3.筛选回收(多个GC线程并行)
